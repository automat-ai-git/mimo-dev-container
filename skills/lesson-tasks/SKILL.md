# Skill: Создание задач для урока из слайдов

## Что делает этот навык

Читает PDF-слайды (или изображения) урока MiMo-Code, анализирует демо-кейсы и домашние задания, затем создаёт соответствующие задачи в Plane.

---

## Шаг 1: Проверка статусов во всех проектах

**Это обязательный первый шаг перед любой работой с задачами.**

1. Получи список всех проектов через Plane MCP (`list_projects`)
2. Для каждого проекта вызови `list_states`
3. Если у проекта нет статусов — создай базовые через `create_state` (рекомендуемые цвета для ai-boss):

   | Название   | group       | color     |
   |------------|-------------|-----------|
   | Backlog    | `backlog`   | `#d9d9d9` |
   | Unstarted  | `unstarted` | `#3f76ff` |
   | Started    | `started`   | `#f59e0b` |
   | Completed  | `completed` | `#16a34a` |
   | Cancelled  | `cancelled` | `#dc2626` |
4. Доложи пользователю: какие проекты уже имели статусы, каким были добавлены

---

## Шаг 2: Чтение слайдов урока

1. Найди материалы урока в папке `sessions/{номер-урока}/slides/`
2. Поддерживаемые форматы: PDF, JPEG, PNG
3. Если страниц больше 20 — читай по частям (по 20 страниц)
4. Ищи:
   - Демо-кейсы (название, папка, ход демо, время)
   - Домашние задания (количество, содержание)

---

## Шаг 3: Анализ папки demo (если есть)

1. Перейди в `sessions/{номер-урока}/demo/`
2. Для каждого демо-кейса прочитай `README.md`
3. Извлеки: название, время, список файлов, шаги демо

---

## Шаг 4: Формирование описаний задач

Plane поддерживает Markdown — используй его для форматирования описаний.

### Демо-кейс

**Название:** `Демо: {название}` — префикс `Демо:` опционален, используй по ситуации

Шаблон описания (Markdown):

```
Подготовка демо-кейса для {краткое описание}.

### Что делать:
- {файлы/данные}
- {что показать}

### Ход демо (~{время} мин):
1. **{Шаг 1}** — {что делать}
2. **{Шаг 2}** — {что делать}

### Папка:
`sessions/{номер-урока}/demo/{имя-папки}/`
```

### Домашнее задание

**Название:** `ДЗ #{номер}: {краткое описание}`

```
{подробное описание задания}

### Что делать:
1. {пункт 1}
2. {пункт 2}

### Подсказки:
- {подсказка 1}
```

---

## Шаг 5: Создание задач в Plane

1. Найди проект нужного урока по названию
2. Используй `create_work_item` с параметрами:
   - `project_id`: ID проекта
   - `name`: название задачи
   - `description`: описание в Markdown
   - `state`: ID статуса **Backlog**

---

## Шаг 6: Проверка результата

1. После создания всех задач вызови `list_work_items` для проекта
2. Если задачи есть в списке — доложи пользователю:
   > «Создано X задач: [список названий]»
3. Если задач нет — попробуй создать без `state` и с упрощённым описанием
4. Снова вызови `list_work_items` — если задачи появились, доложи об этом

---

## Тестовые задачи

Тестовая задача — **валидный инструмент** для проверки работы API или статусов в новом проекте.  
Создавай её явно с названием `Тест: {цель}`, проверяй результат, затем удаляй через `delete_work_item`.

---

## Правила оформления

- Все названия задач на русском языке
- Описания в Markdown
- Префикс `Демо:` для демо-кейсов — опционален
- Домашние задания: префикс `ДЗ #`
- Время демо указывай в минутах: `~15 мин`
- Пути к папкам — относительно корня проекта
